Robust State Feedback

외란에 강인한 제어기 설계(Integral+Anti-Windup)
  • 상태피드백 시 초기 오차누적을 제거하기 위해 발생하는 과도한 Overshoot을 제거하기 위해 적분기+Anti-Windup 기능을 적용한다.
  • 적분기적용 : 기준입력이 있는 상태피드백 제어기에 Integration Action을 추가하여 Plant 외란 입력으로 인해 오차가 발생해도 적분기를 통해 오차를 없애 외란의 영향을 제거한다.
  • Anti-Windup적용 : 또한 적분기에 Anti-Windup 기능을 적용하여 오차누적으로 인해 발생하는 Overshoot를 제거한다.

Closed-Loop System with Reference Input

피드백게인 K 설계
open-loop 시스템 응답특성()이 나쁘므로 피드백게인 K를 설계/적용하여 시스템의 응답특성을 0.1% overshoot, 1s settling time으로 개선하기 위해, 극점을 로 하는 closed-loop시스템()이 되기 위한 피드백게인K를 다음과 같이 설계한다.
roots([1 10 26]) % 특성방정식의 근 확인
ans = 2×1 complex
-5.0000 + 1.0000i -5.0000 - 1.0000i
p_fb=[-5+1i, -5-1i]; % closed-loop 시스템 극점을 원하는 값으로 정의하고,
K_fb = place(A, B, p_fb) % 원하는 극점 p_fb를 갖는 close-loop 시스템의 피드백게인 K를 구함.
K_fb = 1×2
12.9900 -1.0000
기준입력이 도입된 closed-loop Plant정의
% 라플라스 변환을 통해 N_를 구함
N_=-1/(C*(A-B*K_fb)^-1*B)
N_ = 13.0000
% 행렬 역변환을 통해 N_를 구함
row_A=size(A,1); % A행렬의 행값을 return -> A행렬이 3x3 matrix이면 3을 return
Z=[zeros([1,row_A]) 1]; % (1행 row_A열=1x3)인 행렬을 만들고 끝에 1을 추가
N=inv([A,B;C,D])*Z';
Nx=N(1:row_A); % 1행부터 row_A행까지
Nu=N(1+row_A); % row_A+1행
N_=Nu+K_fb*Nx
N_ = 13.0000
A_fb=A-B*K_fb;
B_fb=B*N_;
C_fb=C;
D_fb=0;
closed-loop 시스템 극점(안정도)을 확인한다.
charEq_fb=poly(A_fb);
roots(charEq_fb);
eig(A_fb)
ans = 2×1 complex
-5.0000 + 1.0000i -5.0000 - 1.0000i
closed-loop system step응답 확인
상태피드백을 통해 시스템 응답특성을 개선한 결과 1[V] step입력 시 1[V]로 출력이 되며, 정상상태에 다다르는 시간도 3s에서 1.5s미만으로 단축되었다.
step(ss(A_fb, B_fb, C_fb, D_fb))

Add State Estimation

Integral Action에서 수행한 제어는 직접 센싱을 통해 피드백정보를 획득하여 피드백제어를 수행하였지만, 다음과 같이 관측게인L을 설계/적용하여 관측기 상태추정을 통해 피드백제어를 수행한다.
피드백게인 K를 "Matrix(K*u) (u vector)"로 설정하지 않으면 아래와 같이 게인 2개로 각각 설정해야 함
< 피드백게인 K를 "Matrix(K*u) (u vector)"로 설정안함(좌) vs 피드백게인 K를 "Matrix(K*u) (u vector)"로 설정(우) >
Observability 확인
Qb=obsv(A,C);
rankQb=rank(Qb)
rankQb = 2
관측게인L 설정 (임의의 위치로 State Feedback Gain을 설정)
p_ob=[-6+2i -6-2i];
L=place(A', C', p_ob)';
관측기 상태방정식 정의
Aob=A-L*C;
Bob=[B-L*D L];
Cob=[0 0];
Dob=[0 0];
sys_observer = ss(Aob, Bob, Cob,Dob);
관측기 초기값 설정 후 시뮬레이션을 실행하면, 관측기를 통해 추정한 상태변수를 통해서도 동일하게 Plant 입/출력에 외란인가 시 오차없이 제어입력을 추종하는 것을 확인할 수 있다.

Add Anti-Windup

Closed-Loop System with Anti-Windup

Integrator Windup이란
  • 적분기가 포함된 제어기 출력에 단순이 제한을 걸어 close-loop 성능을 악화시키는 것을 말한다.
  • Integrator Windup으로 인해 Saturation이 발생하면, loop를 open 시켜 시스템을 불안정하게 만든다.
  • 제어기에 적분기가 포함된 상태에서 Saturation이 발생하여 출력제한(Limit, virtually open)이 발생하면, 오차가 누적되는 만큼 적분기가 출력제한을 훨씬 초과하는 제어출력을 생성하기 때문에 시스템은 불안정해 진다.
Anti-Windup이란
  • 기준입력이 클 경우 error를 0으로 빨리 수렴시키기 위해 제어입력이 커지게 된다. 이는 제어기 파손으로 이어질 수 있으므로 기준입력을 허용가능한 제어기 출력으로 제한해야 하는데 이를 Anti-Windup 제어라 한다.
  • Anti-Windup을 적용하기 위해서는 적분기에 Saturation Limit을 걸어야 한다.
  • PID제어기와 같이 적분기가 포함된 제어기의 적분기에 Anti-Windup을 적용하면 액츄에이터 Saturation 발생 시 제어기 불안정성을 방지하고, Saturation이 발생하지 않으면 Anti-Windup은 open-loop가 되어 동작하지 않는다.
< Anti-Windup when Actuator Saturation > < Anti-Windup Open-loop when Actuator No Saturation >
적분기 및 관측기를 적용하여 상태피드백을 수행하는 제어시스템에 Anti-Windup을 다음과 같이 적용한다.
시뮬레이션 실행 결과 Anti-Windup을 적용함으로써 제어출력이 제한되어 시스템 파손을 방지할 수 있다.
즉, Anti-Windup을 통해 Plant(DC모터 상태방정식)입력에 abs(24V)의 Saturation Limit을 걸어 제어출력이 24[V]이상 or -24[V]이하의 전압이 나와도 Plant에는 24[V]이하 or -24[V]이상의 전압이 안가되어 과도한 출력으로 인한 제어기 파손을 방지할 수 있다.
한편 제어출력(모터입력전압)이 제한되는 만큼 응답속도는 늦어진다. 즉, 입력이 500[rad]인 경우 ±3[V]로 출력제한하는 경우보다 ±12[V]로 출력제한하는 경우가 더 짧은시간 이내에 500[rad]까지 이동하게 된다.
