LQG 이론 - Solution to the LQG Problem

Step1: LQR(optimal linear quadratic regulator) Design

LQR 문제는 Pole-Placement와 같이 full state feedback 시스템 설계 시 피드백제어게인 K를 결정하는 방법 중 하나로, 상태피드백시스템 응답이 안정()하고, x와 u에 관한 성능평가함수 를 최소화 하는 피드백게인 를 찾는 것이다.
  • Pole-Placement는 극점을 원하는 곳으로 이동시키는 피드백게인 K를 설계하는데, 극점을 어디에 위치시켜야 하는지가 직관적이지가 않다.
  • LQR은 극점의 위치를 결정하지 않고, closed-loop system의 optimal K를 찾는 방법이다.
  • 즉, LQR은 입력과 상태출력 간의 관계에 대한 성능평가함수인 의 고, P를 통해 제어게인 K를 설계하는 방법이다.
LQR 설계 시 Plant 다음과 같이 정의한다.
Plant에 피드백게인 K를 적용하여 성능평가함수 J의 최적해를 구하는 LQR 제어법칙 를 설계해야 한다.
상태와 입력간의 관계에 대한 성능평가함수 J를 다음과 같이 정의하고 J를 최소화 하는 피드백게인 K를 찾기 위해 algebraic Riccatie Equation 해 를 구한다.
  • 최적제어를 위한 성능평가함수를 통해 R, Q의 관계를 다음과 같이 정의한다.
  • Q(Control Performance)가 커지면, 제어성능이 좋아지고(오차가 0에 빨리 수렴)는 반면 제어입력을 많이 쓰게 된다. -> Q가 커지면 제어입력 게인이 크게 나온다.(이해불가, 확인요???)
  • R이 커지면, 제어입력이 최적이 되어야 하므로 제어성능은 줄어들더라도 제어입력이 작아진다. -> R이 커지면 제어입력 게인이 작게 나온다.(이해불가, 확인요???)
  • 을 알고 있으므로 P는 계산을 통해 구할 수 있다.
P를 구하면 피드백게인 K는 다음을 통해 구할 수 있다.
플랜트에 제어법칙 를 결합하여 K를 통해 상태피드백을 수행하는 Plant를 다음과 같이 재정의한다.

Step2: LQE(Kalman filter - Optimal State Estimator) Design

Process Noise & Measurement Noise가 유입되는 Plant의 경우 Noise를 고려하지 않고 관측기를 설계하면 전상태를 추정할 수 없게 된다. 이 경우 노이즈()가 추가된 Plant를 대상으로 상태를 추정하는 Kalman Filter를 이용하여 관측기를 설계해야 한다.
  • Kalman filter를 적용하기 위해 Plant에 노이즈를 추가하여 Plant를 다음과 같이 정의한다.
  • 피드백을 통해 시스템에 입력되는 제어입력 와 Plant 출력에 노이즈가 추가된 신호를 입력으로 받는 칼만필터를 다음과 같이 설계한다.
  • Kalman filter를 다음과 같이 설계하고 algebraic Riccatie Equation의 해 를 구하고, 를 통해 칼만게인 L을 구다.

  • 제어게인 K가 적절히 설계되더라도 칼만게인 L이 적절하게 설계되지 않으면 LQG closed-loop 시스템은 발산할 수도 있다.

LQR with / without Reference Input

pole-place방법이 아닌 LQR방법을 통해 다음과 같이 최적의 피드백게인을 찾는다. System dynamics가 일 때 lqr함수를 이용하면 다음과 같이 K, S, e를 return한다. 처음에는 를 1로 설정하고, 시뮬레이션 해 가면서 최적의 을 찾는다.
  • ← A, B, Q, R 입력을 통해 P는 자동으로 계산됨
  • ← P를 통해 K를 계산
  • ← 피드백게인 K를 적용한 closed-loop system의 극점확인
% Q=C'*C; R=1;
% Q=[100 0; 0 100]; R=1;
Q=0.05; R=1; % R이 커지면 제어성능은 작아지고, 제어입력이 작아진다.(이해불가, 확인요???)
N=0; % Q가 커지면 제어성능이 크게 나오는 대신 제어입력이 커진다.(이해불가, 확인요???)
[K,P,eig_value]=lqr(A,B,Q*eye(3),R,N)

기준입력 및 상태피드백게인이 적용된 closed-loop 시스템 상태방정식에서 pole-placement를 통해 설계한 게인을 lqr을 통해 설계한 게인 K로 변경만 하면 된다. 이를 위해 기준입력을 추종하는 상태피드백 제어기를 다음과 같이 정의한다.
← .
←
row_A=size(A,1); % A행렬의 행값을 return -> A행렬이 3x3 matrix이면 3을 return
Z=[zeros([1,row_A]) 1]; % (1행 row_A열)인 행렬을 만들고 끝에 1을 추가
N=inv([A,B;C,D])*Z'; % [A,B;C,D]'*[0 1]'
Nx=N(1:row_A); % 1행부터 row_A행까지
Nu=N(1+row_A); % row_A+1행
N_=Nu+K*Nx;
기준입력 및 상태피드백게인이 적용된 closed-loop 시스템 상태방정식
A_lqr=A-B*K;
B_lqr=B*N_;
C_lqr=C;
D_lqr=0;
closed-loop 시스템 극점(안정도) 확인
eig(A_lqr)

-0.3770 + 3.7329i -0.3770 - 3.7329i -1.2788 + 0.0000i

LQG - Full State Feedback by Kalman Filter

LGQ는 칼만필터를 통해 추정한 상태정보를 이용하여 상태피드백을 수행하는 방법으로 kalman filter와 상태피드백 게인K 설계는 서로 영향을 미치지 않기 때문에 독립적으로 수행될 수 있다.
따라서 먼저 상태피드백 게인 K를 구하고, 그 다음 kalman filter를 설계하여 상태를 추정한 다음, 마지막으로 추정된 상태정보와 상태피드백 게인K를 합쳐 상태피드백을 수행한다.
다음 절차를 통해 LQG를 설계한다.
  • 전상태관측기를 칼만필터로 대체한다.
  • 전상태관측기 추정값을 피드백하지 않고, 칼만필터를 통해 예측된 예측값(노이즈를 반영하여 매주기마다 없데이트 됨)을 피드백한다.
  • Kalman Filter Block은 공분산(P_), 칼만게인(K) 등 칼만필터에 필요한 파라미터를 자동으로 업데이트 한다.

Why LQG?

전상태 관측기가 기준입력을 잘 추종하는지 확인한다.
Plant에 노이즈가 추가된 상태에서도 기준입력을 잘 추종하는지 확인한다.
아래와 같이 단순히 전상태관측기를 칼만필터로 대체한 다음 상태피드백을 하면 제어가 안됨(21.04.12)
따라서 LQG기법을 이용해서 칼만게인을 설정해야 함 -> Lecture 23- Optimal LQG Control.pdf, 22p
