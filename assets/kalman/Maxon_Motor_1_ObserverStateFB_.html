State Feedback Control by full state observer(관측기를 통한 전상태모니터링)

close all;clc
상태피드백 제어기 설계(Observer Full State Feedback)
  • place를 이용하여 관측게인(L)을 설정하고, 피드백을 통한 전상태관측기 상태방정식을 설계한다.
  • 불안정한 시스템에 피드백게인을 적절히 설계하여 상태피드백을 수행하면 시스템이 안정해 질 수 있음을 확인한다.
상태추정
  • θ입력을 통해 [i,θ]를 모두 추정하고, 전달함수의 전류/속도 출력과 전상태관측기 추정결과를 비교한다.

Plant 상태방정식 정의

Plant 정의

다음과 같이 Plant를 정의한다.
% Ra = 0.299; % [Ω]
% La = 0.000082; % [H]
% Kt = 0.0302; % [Nm/A]
% Kb = 0.0301; % [V/(rad/s)]
% Jm = 0.0000142; % [Kg*m^2] <- No load
% Bm = 0.0000052123; % [Nm*(s/rad)] <- No load
%
% A=[-Ra/La -Kb/La 0;
% Kt/Jm -Bm/Jm 0;
% 0 1 0];
% B=[1/La 0 0]';
% C=[0 0 1];
% D=0;
A=[-0.4 0 -0.01; 1 0 0; -1.4 9.8 -0.02];
B=[6.3; 0; 9.8];
C=[0 0 1];
D=0;
% Plant 초기값 설정
X0=[0 0 0]; % Plant 상태변수 초기값

안정도를 확인해 보면 eigenvalue가 positive로 시스템은 불안정한 상태이고, step 응답을 통해 출력을 확인해 보면 다음과 같이 발산함을 알 수 있다.
eig(A) % Plant 극점(안정도) 확인
ans = 3×1 complex
-0.6565 + 0.0000i 0.1183 + 0.3678i 0.1183 - 0.3678i

이산시스템 응답

상태방정식을 이산화하고, open-loop system의 step응답을 확인하면 다음과 같다.
step(ss(A,B,C,D))
hold on;
sys = ss(A,B,C,D);
dt=1; % sampling 주파수를 1s로 크게해서 이산화 여부 확인
d_sys = c2d(sys,dt);
step(d_sys)

관측게인(L)을 포함하는 관측기 상태방정식 설계

다음과 같은 이유로 전상태관측기를 설계해야 한다. (단, 노이즈를 고려할 경우 전상태관측기 적용은 불가능하고, 칼만필터 추정기가 사용되어야 함)
  • 상태피드백을 위해서는 제어피드백게인 K를 설계하여 극점을 원하는 곳으로 이동시켜야 한다.
  • 전상태측정이 가능한 경우에만 pole-placement를 통해 피드백게인 K 설계가 가능하다.
  • 여기서 설계된 전상태관측기 모델이 칼만필터에 적용된다.
Qb=obsv(A,C); % 가관측성 확인
rankQb=rank(Qb) % rank=2(Fully Observable): 전상태 관측가능
rankQb = 3
% 관측게인 L설정 (임의의 위치로 State Feedback Gain을 설정)
% p_ob=[-50+50*i -50-50*i];
p_ob=[-500 -50+50*1i -50-50*1i];
L=place(A', C', p_ob)';
% 관측게인을 포함하는 전상태관측기의 상태방정식 정의
Aob=A-L*C;
Bob=[B-L*D L];
Cob=eye(3);
Dob=[0 0; 0 0; 0 0];
sys_observer = ss(Aob, Bob, Cob,Dob);
X0ob=[10; 0; 0]; % 관측기 상태변수 초기값
관측기 안정도를 확인한다.
eig(Aob)
ans = 3×1 complex
102 ×
-5.0000 + 0.0000i -0.5000 + 0.5000i -0.5000 - 0.5000i

제어 가능한 경우에만 제어기 설계가 가능하므로 시스템의 가제어성(Controllability)을 확인한다.

rankQc = 3

기준입력을 포함하는 상태피드백 설계

게인을 무한히 키우면 응답속도는 빨라지나 노이즈/외란과 같은 잡음성분 또한 커지게 되어 시스템 출력에 영향을 미치게 된다. 따라서 게인을 왼쪽에 얼만큼 설정해야 하는지는 여러 요인을 고려해서 통합적으로 설계해야 함. (최적설계방법:LQR 설계와 관련있음)
p_ct=[-15 -10+10*1i -10-10*1i];
K=place(A,B,p_ct)
K = 1×3
-70.0116 117.4274 48.5360
Step응답을 확인하기 위해 기준입력에 대한 정상상태 최종값을 로 정의하고, 전상태피드백을 이용한 제어법칙에 다음과 같이 기준입력을 도입한다.
→
의 어떠한 값에 대해서도 가 되도록 하기 위해 라 하면 는 다음과 같이 구할수 있고,
→
따라서 기준입력이 포함된 제어입력은 다음과 같다.
row_A=size(A,1); % A행렬의 행값을 return -> A행렬이 3x3 matrix이면 3을 return
Z=[zeros([1,row_A]) 1]; % (1행 row_A열)인 행렬을 만들고 끝에 1을 추가
N=inv([A,B;C,D])*Z'; % [A,B;C,D]'*[0 1]'
Nx=N(1:row_A); % 1행부터 row_A행까지
Nu=N(1+row_A); % row_A+1행
N_=Nu+K*Nx;
상태방정시 하나로 기준입력 및 상태피드백게인이 적용된 전체 closed-loop 시스템 상태방정식을 표현할 경우
→
A_fb=A-B*K;
B_fb=B*N_;
C_fb=C;
D_fb=0;
closed-loop 시스템 극점(안정도) 확인
eig(A_fb)
ans = 3×1 complex
-15.0000 + 0.0000i -10.0000 +10.0000i -10.0000 -10.0000i

결과확인

기준입력 및 상태피드백을 시스템에 적용한 결과, 1[V] step입력 시 1[V]로 출력되는 것을 확인할 수가 있다. 하지만 상태피드백게인이 크기 때문에 step입력 초기 응답 시 과도한 Overshoot 발생하게 된다. (Overshoot 방지를 위해 Anti-Windup이 적용되어야 함)
step(ss(A_fb,B_fb,C_fb,D_fb))

전상태관측기 설계

전상태피드백 제어는 시스템 출력이 기준입력을 따라가게 된다.
open([getenv('sldir') 'StateFeedback\Maxon_Motor_1_ObserverStateFB_3.slx']);
대부분의 경우 모든 상태변수가 측정되지 않기 때문에 관측기를 통해 상태변수를 추정해야 한다.
관측 가능한 경우에만 관측기 설계가 가능하므로 시스템의 가관측성(Observability)을 확인한다. 상태방정식의 상태변수가 3개일 경우 "rank=3"이면 가관측하다.
Qb=obsv(A,C);
rankQb=rank(Qb) % rank=3(Fully Observable): 전상태 관측가능
rankQb = 3
관측기응답이 제어기응답보다 빨라야 하므로, 관측기극점이 제어기극점보다 크게 설정되어야 한다. (제어기극점보다 관측기극점이 극좌표 좌반면에 위치해야 함, 일반적으로 5~10배(시뮬레이션 확인요))
% 관측기 게인 L설정 (임의의 위치로 State Feedback Gain을 설정)
p_ob=[-50 -60+60*1i -60-60*1i];
L=place(A', C', p_ob)';
% 관측기 상태방정식 정의
Aob=A-L*C;
Bob=[B-L*D L];
Cob=eye(3);
Dob=[0 0; 0 0; 0 0];
sys_observer = ss(Aob, Bob, Cob,Dob);
% 관측기 극점(안정도) 확인
eig(Aob)
ans = 3×1 complex
-50.0000 + 0.0000i -60.0000 +60.0000i -60.0000 -60.0000i
시뮬레이션 결과 Plant출력값(θ)과 관측기추정값()이 일치하며, 초기 과도한 overshoot이 발생하기는 하나 기준입력을 잘 추종하고 있음을 확인할 수 있다.
하지만 단순히 상태피드백을 통한 제어를 수행할 경우 초기 과도한 Overshoot 발생, 외란 또는 노이즈로 인한 제어성능저하가 발생하므로 제어기법을 추가로 적용해야 한다.
  • 초기 과도한 overshoot이 발생하는 경우 -> Anti-Windup 적용
  • 외란 입력으로 제어성능이 나빠지는 경우 -> Robust State Feedback을 수행해야 함.
  • 노이즈 입력으로 제어성능이 나빠지는 경우 -> LQG(state estimation by kalman filter)를 수행해야 함.

전상태관측기에서 노이즈 고려 -> Kalman Estimator가 됨

LQR - 전상태관측기를 통한 상태피드백 제어시스템 설계 (노이즈에 취약함)
  • place를 이용하여 피드백게인(K)를 설정
  • 기준입력 도입을 위한 피드포워드게인(N_) 설정
  • 기준입력에 피드포워드게인(N_)을 추가, 전상태관측기의 피드백정보에 피드백게인을 추가하여 상태피드백 제어 시스템을 구성한다.
시뮬레이션 검증
  • 노이즈가 없는 상태에서 상태피드백을 통해 Plant 출력이 reference input을 잘 추종하는지 확인한다.
  • 전상태관측기는 노이즈가 고려되지 않았기 때문에 Plant에 노이즈(Process/Measurement)가 추가된 상태에서는 상태를 추정할 수 없음을 확인한다.
Plant에 노이즈(Process-모델링오차 or Measurement-센서측정오차) 유입 시 시스템 출력
Continuous Plant가 다음과 같을 때
kalman estimator(칼만추정기)는 continuous 또는 discrete estimation problem에 대한 최적의 해를 찾아준다. 즉, 정상상태 오차 분산을 최소화 하여 상태를 추정한다.
